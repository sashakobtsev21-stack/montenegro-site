Владелец несколько раз присылал фото для сайта; часть оказывалась **скачанной из сети/стоков** (включая файл Shutterstock), иногда переименованной в 001–007, чтобы замаскировать происхождение. Такое публиковать НЕЛЬЗЯ (чужой копирайт, §18 — иск/штраф летит на домен владельца). Я отказывался даже по прямой просьбе «загружай эти» и «сделай вид, что мои».

**Как отличать (проверять ВСЕГДА перед публикацией):** прогнать `sharp.metadata()` по файлам (см. одноразовый скрипт-хелпер, который я пишу в `scripts/_imgmeta.mjs` и удаляю после).
- **Из сети/сток** = мелкое разрешение (~800–2048 px, ровно 1000/1024/1280 по ширине), **EXIF вырезан**, имена типа `shutterstock_*`, `*-min-1024x577`, `caption (N)`, `Things-to-do-*`. → отказать, предложить CC с Wikimedia или свои оригиналы.
- **Своё** = большое разрешение, **EXIF камеры** (Apple/iPhone/Samsung…), имена `IMG_xxxx`. У владельца есть iPhone: рабочая выгрузка пришла из папки `iCloud Photos` (~2048px, Apple EXIF, `IMG_0979.JPEG`…). Такое — публикуем.

**Свои фото → процесс:** конвертировать в webp ≤200КБ (`sharp` .rotate().resize(width 1280)), класть в `public/images/{slug}/gN.webp`, credit `'Фото: Georgia Guidebook'`, и включать у статьи `verifiedAt` + `verifiedPhoto` → фирменный бейдж **«Проверено»** (это и есть выплата за свои снимки, см. [[site-structure-changes-2026-06]]). Первый пример — Кутаиси (`3160fca`). Конвейер CC-фото — в [[photo-pipeline-hybrid]].

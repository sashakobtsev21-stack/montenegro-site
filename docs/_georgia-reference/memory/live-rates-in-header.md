Живой блок «курс лари + погода» владелец захотел в ГЛОБАЛЬНОЙ шапке, а не на главной. Финальная реализация: `src/components/RatesBar.astro` — тонкая utility-полоса в самом верху шапки (курс USD/EUR/RUB/UAH инлайн + «Погода» в нативной `<details>`-выпадашке: Тбилиси/Кутаиси/Батуми/море). Данные — build-снапшот `src/lib/liveSnapshot.ts` + `public/js/live-data.js` по `data-live` ключам (мемо: 1 fetch на билд). `LiveData.astro` удалён.

**Why:** владелец подряд отверг 4 варианта на главной (центр-блок → «всё в столбик» → «сбоку») как некрасивые; курс полезен на любой странице → его место в хедере.

**How to apply:** не возвращать большой live-блок на главную без явной просьбы; новые «живые» индикаторы делать компактно в шапке. По UI-правкам этого владельца лучше показывать превью-опции до реализации. Связано с [[site-structure-changes-2026-06]].

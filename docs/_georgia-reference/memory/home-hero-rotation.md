Главная (`src/layouts/HomePage.astro`) показывает hero-фон, который меняется **каждые 3 часа**: инлайн-скрипт берёт `floor(Date.now() / 10800000) % heroVariants.length` → ставит `srcset`/`src` на `<img id="heroBg">` и обновляет кредит. Чистая статика, без бэкенда (сайт = SSG, без SSR/БД). Владелец (2026-06-16) попросил именно ротацию вместо одной обложки.

**Набор (2026-06-19, владелец отобрал из подборки ~20 Unsplash, кадры 1–4 отверг) — 5 ЯРКИХ кадров Unsplash** (исходники `src/assets/hero/*.jpg` 2000px; **2026-06-20: `<picture>` — AVIF до 1600px (per-image quality `q` в `heroSources`: детальный Гергети q40, мягкая Кахетия q62) + webp-фолбэк до 1280px q64 для ~5% браузеров без AVIF**. Раньше был только webp кап 1280 → на десктопе фон апскейлился до ~1920 и МЫЛИЛ (жалоба владельца 2026-06-20); AVIF жмёт плотнее, поэтому 1600px влезает в ≤200КБ там, где webp нет. На FHD-десктопе апскейл стал 1.2x вместо 1.5x — заметно резче): **gergeti-kazbek-2** (Deogyeon Hwang, ДЕФОЛТ), **ushguli** (Patrick Schneider), **mestia-ushba** (Mike Swigunski), **tbilisi-colorful** (Inna Nasonova), **kakheti-vineyards** (Giga Goriashvili). Все — Unsplash, кредит формата «Автор · Unsplash» + href на страницу фото. **Batumi-coast (высотки Орби) удалён по просьбе владельца 2026-06-19** (не нравятся новостройки на главной). Прежний набор (5 CC-кадров Commons: gergeti-kazbek/tbilisi-oldtown/mtskheta-jvari/tbilisi-sunset/kakheti-signagi) заменён целиком.

**УРОК (2026-06-16, дважды правил):** для hero нужны ЯРКИЕ кадры. Ночные кадры под тёмным скримом `.hero__scrim` выглядят «серыми/мутными» — владелец дважды жаловался («серое некачественное фото»). И разрешение ≥2400px + q≥66, иначе на десктопе мыло (1600px/q56 было мало). НЕ ставить ночные кадры и не ужимать ниже 2400.

**УРОК-2 (2026-06-17, владелец: «фото тусклое»):** прежний скрим был ВЕРТИКАЛЬНЫМ и тяжёлым (slate 20→44→74% сверху вниз) — глушил всё яркое фото (~−35% яркости, замерил рендером: ≈RGB100 vs source ≈148). Сделал скрим **НАПРАВЛЕННЫМ**: тёмный СЛЕВА (под левым текстом) → почти прозрачный СПРАВА → правая часть фото яркая/«праздничная» (правый край вернулся к ≈RGB158). Токены `--hero-scrim-text/-2/-3` (левый градиент) + `--hero-scrim-base` (лёгкая подложка снизу под кредит); мобайл (≤768px) — лёгкий вертикальный (10/42/60). Читаемость белого заголовка держит УСИЛЕННАЯ двойная `--hero-text-shadow`. Контраст проверен рендером 1280/390px. Коммит `58a4b62`. **Правило: не затемнять ВСЁ фото ради контраста — затемнять только зону под текстом.** **(2026-06-20, UX-аудит):** подзаголовок и `.search__hint` уже были `--hero-text-muted` (white 88%), а НЕ серые (агент ошибся) — реальная причина мыла на светлых кадрах (снег/закат) = слабый скрим; усилены токены `--hero-scrim-text/-2/-3/-base` (60/34/6/34 → 70/50/16/40) и мобильный скрим (10/42/60 → 20/50/66), правую/яркую часть фото не трогали.

**Дефолтный кадр (variant 0) рендерится СЕРВЕРНО** на `<img src/srcset>` → яркое фото видно сразу, даже если JS/CSP/кэш подведут (после фикса серого фона). Ротация (`hero-rotate.js`) — прогрессивное улучшение поверх.

**Как поменять набор:** положи исходник (≤2000px jpg) в `src/assets/hero/`, импортируй и добавь объект в массив `heroSources` (`img`/`credit`/`href`/`focus`; `credit`=«Автор · Unsplash» или «… · Commons», `href`=страница фото). Кредит под фото меняется вместе с кадром; локализуется только префикс `home.photoPrefix` («Фото»/«Photo»). Отбор кадров — я «глазами» из контактных листов (Unsplash `unsplash-candidates.mjs`/`build-unsplash.mjs`); владелец отбирает финал из подборки и может попросить заменить любой. **Грабля: каждый вариант — И AVIF, И webp (в `dist/_astro`) — ≤200КБ, иначе `npm run qa` = NO-GO (жёсткий гейт, проверяет и подключённые из `<source>`/`<img>`). Формат: `AVIF_WIDTHS=[640,960,1280,1600]` per-image `q`, webp-фолбэк `WEBP_WIDTHS=[640,960,1280]` q64. Детальные снежные горы (gergeti/mestia) на 1600 webp >200КБ — потому и AVIF. Менять набор: задать `q` под кадр и сверить вес после build.**

**Грабля (2026-06-20): Read НЕ рендерит `.avif` (отдаёт бинарь в контекст — засоряет). Чтобы оценить AVIF глазами — конвертни в png: `sharp(x).png().toFile(...)` → Read png.**

**Перф (§15) — важные грабли:**
- Исходники надо ужать до **≤1600px по ширине**, иначе `astro:assets` дополнительно генерит тяжёлый native-вариант (нативный 5248px → webp 715КБ). Скрипты `_scratch/resize-hero.mjs` (resize ≤1600) и `_scratch/crop-hero.mjs` (сверхширокие панорамы Вардзия/Мцхета → кроп 16:9) через sharp. Итог: мобайл-вариант 35–144КБ.
- LCP: `<img>` рендерится **без src**, инлайн-скрипт сразу за ним ставит srcset выбранного кадра (без двойной загрузки), `fetchpriority=high`; `<noscript>` — дефолтный кадр; `.hero` имеет `background-color: var(--color-slate)` как заглушку до загрузки/без JS.

**Скачивание с Wikimedia Commons:** не burst'ом (ловит HTTP 429) — по одному с паузой ~5с и UA с контактом (`GeorgiaGuidebook/1.0 (https://georgiaguidebook.com; info@georgiaguidebook.com)`).

**Грабли CSP (2026-06-16) — важно:** интерактив на сайте только внешними `/js`-модулями (`script-src 'self'`, без `unsafe-inline`). Инлайн-скрипт (`define:vars`/IIFE прямо в .astro) на проде Cloudflare блокируется → hero был пустой (в `dev`/`preview` CSP не применяется, баг при сборке не виден!). Поэтому логика ротации вынесена в `public/js/hero-rotate.js`, а данные кадров — в `<script type="application/json" id="hero-variants">` (data-блок, не исполняется → CSP-безопасен, как JSON-LD). Никогда не возвращать инлайн-JS.

**Визуальный аудит (мобайл/hero) — как я проверяю без ручного браузера:** надёжнее всего `chrome --headless --screenshot --window-size=WxH --virtual-time-budget=6000 URL` (рендерит И снимает на нужной ширине; dist отдаю node-статиком на :4321) + нарезка sharp'ом на сегменты для детального просмотра. Грабли: `_scratch/shot.mjs` (CDP-эмуляция мобайла 360/320, DPR2) даёт ТОЧНУЮ ширину, но Node-встроенный WebSocket рвёт CDP-сокет (close 1006) — для рендера всё равно используй `--headless --screenshot`. `astro check`/eslint спотыкаются о временный профиль Chrome в `_scratch/` → `_scratch` добавлен в exclude `tsconfig.json` и в `globalIgnores` eslint. Overflow меряю оверлей-скриптом (scrollWidth vs clientWidth) на рендере 360.

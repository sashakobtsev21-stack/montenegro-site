Главная редизайн (2026-06-16, коммит `a73dff4`, решение владельца).

**Что сделано:**
- Блок «С чего начать» (5 плиток-входов) на главной ЗАМЕНЁН на **«Витрина Грузии»** — золотая лента-карусель, медленный авто-скролл справа налево. Компонент `src/components/ShowcaseRail.astro`. Навигация по разделам НЕ потеряна — она в шапке/подвале (5 PRIMARY_TILE_KEYS + вторичные).
- Под витриной — заметный **бэнд «Услуги для жизни в Грузии»** (вино-фон, солнечный CTA → `/relokatsiya/uslugi/`), поднят с хаба релокации на главную как продаваемое рекламное место. Текст берётся из `dict.relocation.uslugi` (heading/text/cta) — отдельных строк не заводил.
- Витрина = продаваемый инвентарь (§16): наполнена 7 нашими ключевыми материалами, каждая карточка в **золотой рамке**. **Открытое рекламное место «Разместите своё заведение» УБРАНО с витрины (решение владельца 2026-06-18, коммит `41430f4`)** — поддержка ad-слота в ShowcaseRail/i18n (`adTitle`/`adKicker`) оставлена спящей; НЕ возвращать без нового решения владельца.

**Как устроено (важно для правок):**
- Карусель — **чистый CSS, без JS** (CSP §18: script-src 'self', инлайнов нет). Бесшовный цикл: лента дублируется, клон `aria-hidden` + `tabindex=-1`. Анимация `transform: translateX(0→-50%)`, ячейки несут `margin-inline-end` (а не gap), чтобы -50% = ровно половина ленты.
- Доступность (§6, WCAG 2.2.2): пауза по `:hover`/`:focus-within` + явная кнопка-пауза (CSS-чекбокс, `:checked ~ … animation-play-state: paused`). При `prefers-reduced-motion` — анимация off, клоны `display:none`, лента `overflow-x:auto` (ручная прокрутка). Глобальное правило reduce в global.css глушит анимации `!important` — поэтому marquee и так встаёт.
- Items курируются в `HomePage.astro` (`showcasePicks`: `{kind:'article'|'route', category?, slug, kicker}`), обложка/заголовок берутся из коллекции (реальные данные, не выдумываем). Маршруты — отдельная коллекция `routes`. Открытое рекламное место (ad-слот) РАНЬШЕ добавлялось отдельным push в `showcaseItems` — сейчас УБРАНО (см. выше); ветка рендера `item.ad` в ShowcaseRail спящая (вернуть = снова push `{ad:true,...}`).
- **Пин-чип города (2026-06-18, `ab57e5d`):** для заведений в витрине — поле `ShowcaseItem.city` → пин-чип «📍 Город» поверх обложки (низ-лево, иконка map-pin + полупрозрачная тёмная подложка для контраста, как hero-кредит; без JS). Рисуется ТОЛЬКО при заданном `city`. Задаётся явно у pick в `HomePage.showcasePicks` (`city:'Тбилиси'`); у контентных карточек (города/маршруты) не задан. Выбор владельца — вариант «пин на фото» (vs kicker/подзаголовок).
- **Золото вынесено в токены** (`src/styles/tokens.css`): `--color-gold`/`--color-gold-light`/`--color-gold-deep`/`--gold-frame` (металлик-градиент)/`--gold-glow`. Единый маркёр платного/партнёрского места (schema `services` уже это подразумевала: «золотая рамка — маркёр спонсора»). Хардкод hex в компонентах по-прежнему запрещён — только эти токены.

**ИСКЛЮЧЕНИЕ из SPEC.** SPEC §10 анти-паттерны прямо запрещают «карусели, автоплей». Витрина — **санкционированное владельцем исключение** (SPEC §8.4 + §10, 2026-06-16): единственный допустимый медленный авто-карусель. Любая новая карусель должна держать те же условия (пауза hover/focus + кнопка, reduced-motion off, без JS) и быть согласована.

**Ловушка слугов:** имя файла ≠ slug. `articles/*/borjomi-gorod.md` имеет `slug: 'borjomi'`; слуги вроде `batumi`/`tbilisi` повторяются в разных категориях (goroda и arenda-avto). Курировать витрину/подборки только по паре **category+slug**, а slug сверять во frontmatter, не по имени файла.

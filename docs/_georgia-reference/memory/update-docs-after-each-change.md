Владелец (2026-06-16): после КАЖДОЙ доработки обновлять все документационные файлы под новое состояние — не давать докам отставать от кода.

**Why:** проект ведётся хендофф-документами (PROGRESS/HANDOFF/SPEC) между чатами и ПК; устаревший роадмап → потеря контекста и дублирование работы.

**How to apply:** в конце задачи обновить `PROGRESS.md` (роадмап/статус + дата), `HANDOFF.md` (снимок: последний коммит, число страниц, «где остановились»); при изменении продукта/правил — `SPEC.md`/`CLAUDE.md`; `AUDIT.md` — отметить закрытые находки; авто-память (новые/изменённые факты) + её ЗЕРКАЛО `docs/memory/`. Проще всего синхронизировать зеркало одной командой — скопировать всю папку памяти в `docs/memory/` (`cp …/memory/*.md docs/memory/`), чтобы индекс и файлы совпадали. Затем коммит+пуш. Связано с [[always-commit-push-main]] и [[article-content-standards]].

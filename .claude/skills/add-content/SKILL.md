# /add-content — добавить материал по конвейеру

Цель: довести новый материал до состояния «можно публиковать» с соблюдением железных правил (SPEC.md → CLAUDE.md). **Не выдумывай факты** (правило 4) и **не публикуй без фото** (правило «фото у всего контента»). Источник правды по продукту — SPEC.md; правила репозитория — CLAUDE.md; статус — PROGRESS/HANDOFF/ROADMAP.

## Аргументы
`<тип> <slug> [тема/запрос]` — тип: `article|news|route|restaurant|service|city`. Если не задано — спроси у владельца тип, slug (латиница-дефис, стабилен навсегда) и есть ли бриф `briefs/<slug>.md`.

## Шаги (выполняй по порядку, не пропуская гейты)

1. **Бриф и факты.** Прочитай `briefs/<slug>.md`, если есть. Нет брифа или фактов (цены/адреса/часы/телефон) — НЕ выдумывай: ставь `<!-- TODO: факт от владельца -->` и перечисли пробелы владельцу. Для общих сведений допустим формат «ориентир/уточняйте» + источник.

2. **Скелет.** `npm run new -- <тип> <slug> --title "…"` (для article можно `--category <cat>`, для restaurant/service `--city "…"`). Создаст ru/uk/en + `public/images/<slug>/` + DoD-чеклист в комментарии. Скелет — `draft:true`.

3. **Черновик ru.** Заполни ru-версию по шаблону §8: H2/H3 под вопросы читателя, добавленная ценность §15, БЕЗ FAQ (убран). Делегируй `seo-content-writer` (Agent) или пиши сам по его правилам. ≥2 внутр. ссылки + хаб раздела. 1–3 AffiliateBox только через `/go/{partner}?c=<slug>`.

4. **Фото (ОБЯЗАТЕЛЬНО).** Гибрид (память photo-pipeline-hybrid): `photo-researcher` (Agent) гонит `scripts/commons-candidates.mjs` → кандидаты CC в `public/images/_scratch/<тема>/`; **ты сам отбираешь глазами** (Read рендерит картинки) и собираешь `scripts/build-gallery.mjs` (манифест `[{url,out,cover?}]`) → webp ≤200КБ. Минимум: cover (CC → `coverCredit` автор+лицензия / своё фото). Своё фото владельца — проверь метаданные (память photo-provenance-check). Для маршрута cover обязателен; для статьи цель — cover + ≥3 инлайн ИЛИ ≥6 gallery.

5. **Переводы.** `uk-translator` → uk (EKTA, сухопутный въезд через Сарпи), затем `en-translator` → en (визы/въезд в общем виде, USD/EUR-ориентир). Паритет: тот же slug, совпадение числовых полей, те же `gallery` src.

6. **Фактчек → приёмка (именно в этом порядке).** `fact-checker` сверяет черновик с брифом (ловит выдуманные факты) → `content-editor` финальный gate (DoD, §15, SEO §14, /go/ + rel, доступность).

7. **Маршрут — геометрия.** Если route и заполнены `stops` → `node scripts/build-route-geometry.mjs` (запечь дорожную линию OSRM).

8. **Гейты (все зелёные перед коммитом):** `npm run check` · `npm run build` · `npm test` (enums+parity+photos) · `npm run test:links` · `npm run lint`. Сними `draft:false` только когда DoD выполнен и фото на месте.

9. **Коммит + пуш в main** (стандартная авторизация, память always-commit-push-main): пара/тройка одним коммитом. Затем **синхронизируй доки** (правило update-docs-after-each-change): PROGRESS/HANDOFF (+ ROADMAP/SPEC/CLAUDE при изменении правил) + авто-память и её зеркало `docs/memory/`.

## Запреты (см. CLAUDE.md «Чего НЕ делать»)
Без React/CMS/БД/платных сервисов. Без хардкода hex (только токены). Без прямых партнёрских URL. Без массовой генерёнки без уникальной фактуры (§15). Сток/из сети как «своё» фото не выдавать. Половину языковой пары не публиковать.

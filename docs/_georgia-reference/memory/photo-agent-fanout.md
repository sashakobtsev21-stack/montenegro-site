Владелец заказал «фото во ВСЕ статьи, максимум, узнаваемые обложки городов» и выбрал режим **полный автомат + мой QA** (а не гибрид). Рабочая схема:

**Агенты (general-purpose, по 1 статье, параллельно ~5):** читают пару ru+uk → `commons-candidates.mjs "Category:<X>"` → отбирают → пишут манифест → `build-gallery.mjs` (webp ≤200КБ) → дописывают frontmatter `gallery` + инлайн `<figure>` в тело (с `width/height` из вывода build-gallery — я добавил туда печать WxH) → ru+uk идентично. НЕ коммитят, build не запускают, обложку достопримечательностей не трогают (обложки городов меняю я).

**Рецепт v2 (снижает мой QA):** галерея и инлайн — НЕПЕРЕСЕКАЮЩИЕСЯ множества; жёсткий скип по заголовку: карты/планы/SVG, гербы, гравюры/архив, портреты/люди, **таблички/щиты/UNESCO-стенды** (частый провал!), сканы, ночные/тёмные, ч-б, вертикальные селфи; ≥1500px, дневной свет.

**Мой QA ОБЯЗАТЕЛЕН** (агенты не видят картинок, ~70% попаданий): генерю превью `node scripts/_qa_prev.mjs <dir...>` → смотрю Read'ом → удаляю файл + правлю ru+uk. Частые промахи: обложка выбрана по имени файла = мусор (забор, ржавая крыша, **инфо-табличка** «ArmAg/Svetitskhoveli» = щит ЮНЕСКО); категория Commons содержит не тот объект (у «Ananuri» — заброшенный особняк и статуя, alt не совпадал с фото); тёмные интерьеры; бытовые детали (родник). Обложку города бери САМ глазами из скачанных кадров (пересобрать cover из URL с `"cover":true`).

Прогресс/конвейер — в [[photo-pipeline-hybrid]]; формат инлайн+галерея — [[city-inline-photo-format]]; свои фото владельца — [[photo-provenance-check]]. `_scratch/` и `scripts/_*.mjs` теперь в .gitignore.

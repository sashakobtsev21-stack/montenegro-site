Требования владельца к статьям контента (применять ко всем новым и при доработке существующих):

- **Фото обязательны.** Обложка-фото у КАЖДОЙ статьи. Галерея в теле: ≥5 фото в «Что посмотреть» и «Маршрутах», ≥10 в «Городах»; в маршрутах — фото к каждому пункту/остановке; в городах фото лаконично интегрированы в каждый абзац.
- **Объём/факты.** Достопримечательности: +3–5 предложений к текущему + интересные факты (2–3 предложения, только если реальные есть). Города: +10 предложений к текущему. Факты — только из источников, не выдумывать.
- **«Что посмотреть»:** раскрывающийся блок «Как доехать из Тбилиси / Кутаиси / Батуми» в каждой статье.
- **CoordCopy** — компактная, рядом с картой.

**КРИТИЧНЫЙ CAVEAT (не нарушать):** фото — ТОЛЬКО CC с Wikimedia (с атрибуцией автор+лицензия, §18) ИЛИ собственные фото владельца. Произвольные фото из Google/веба — нельзя (копирайт). Где CC-фото не хватает до 5/10 — ставить сколько реально есть + `<!-- TODO: фото от владельца -->`. Собственные фото владельца дают и больше фото, и право ставить бейдж «Проверено» (иначе «Обновлено»). Связано с [[accurate-ui-instructions]].

**ОБНОВЛЕНИЕ 2026-06-14:** формат фото изменён — ключевые снимки теперь ВСТРОЕНЫ в текст с подписями (инлайн-фигуры), а нижняя галерея осталась «мини». Подробно — [[city-inline-photo-format]]. Также согласовано будущее правило «цена+режим работы у каждого места» — [[entertainment-section-plan]].

**УТВЕРЖДЁННЫЙ ФОРМАТ СТАТЬИ (Фаза 1, владелец одобрил «делаем так на все», 2026-06-13).** Каждая статья «Что посмотреть»/«Города»/«Маршруты» строится так:
- **Фото-галерея** — компонент `PhotoGallery` (сетка 2/3 кол., lazy, аспект 4:3, лайтбокс public/js/lightbox.js, figcaption с CC-атрибуцией обязательна). Достопримечательности/маршруты ≥5 фото, города ≥10. Обложка обязательна.
- **Google-карта** — компонент `GoogleMap`: Google iframe ленивый ПО КЛИКУ (output=embed, без API-ключа, ничего от Google в статике → без cookie-баннера). На страницах статей Leaflet НЕ используется (остался только в CatalogMap и RoutePage). CSP: `frame-src google.com maps.google.com`.
- **Координаты** — `CoordCopy variant="compact"` в баре под картой (не в шапке) + ссылка «Открыть в Google Картах».
- **«Как доехать»** (только «Что посмотреть») — компонент `AccessFrom`, native `<details>`, из Тбилиси/Кутаиси/Батуми (км+время). Поле frontmatter `accessFrom`.
- **Факты** — 2–3 выверенных интересных факта вплетены в тело (источник в sources, не выдумывать).
- Схема: `articleBase` имеет опциональные `gallery` ({src,alt,credit}) и `accessFrom`. i18n: namespaces map/access/gallery.
- Порядок в ArticlePage: тело → галерея → карта+координаты → «как доехать» → AffiliateBox → FAQ.
- Эталон-образец: `src/content/articles/{ru,uk}/gergeti.md` + `public/images/gergeti/g1..g7.webp`.

Карта по клику оставлена осознанно (бесплатно/приватно). Сразу-видимая Google-карта потребует API-ключ владельца + cookie-баннер — пока не делаем.

**ОБНОВЛЕНИЕ 2026-06-16 (владелец, акцент).** При добавлении ЛЮБОЙ статьи/раздела — только САМЫЕ ЛУЧШИЕ фото (отбор глазами, слабые/мусорные кадры не ставить) и РАЗВЁРНУТОЕ описание ПО АНАЛОГИИ с уже опубликованными: тот же объём и структура (инлайн-фигуры с подписями + мини-галерея, выверенные факты с источником, «как доехать», visit-блок). Без тонких «заглушек»; качество новых материалов держать на уровне текущего эталона. Синхронизировать доки после доработки — [[update-docs-after-each-change]].

Владелец (2026-06-17): **у КАЖДОГО материала на сайте — статья, новость, заведение «Еда»/«Услуги», маршрут, город — ОБЯЗАТЕЛЬНО должно быть фото** (минимум `cover`, лучше ещё `gallery`/инлайн-фигуры). Без исключений. Контент без фото не публикуется и считается дефектом — проверять и при добавлении нового, и ретроспективно по всему сайту.

**Почему:** сайт визуальный/travel; материал без фото выглядит незавершённым и «дешёвым», отталкивает, роняет доверие и поведенческие факторы (→ SEO). Владелец отдельно ловил отсутствие фото (новость про страховку — без cover) и считает это критичным.

**Как применять:**
- При добавлении/правке ЛЮБОГО контента — сначала убедиться, что есть `cover` (и по возможности `gallery`/инлайн). Нет фото → НЕ коммитить: найти CC/своё фото (конвейер [[photo-pipeline-hybrid]], [[photo-agent-fanout]]) и добавить.
- Фото — только CC (Wikimedia: автор+лицензия в `coverCredit`) или свои; не сток/не из сети ([[article-content-standards]], [[photo-provenance-check]]).
- Закрепить машинно: добавить в `scripts/check-parity.mjs` (или отдельный `check-photos`) проверку, что у каждого материала коллекций есть `cover` → сборка/гейт падает при материале без фото. До этого — аудит грепом по `src/content/**`.
- Демо/примеры — ТОЖЕ с фото (выглядят как настоящие места, [[live-site-no-dev-placeholders]]).

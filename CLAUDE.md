# CLAUDE.md — проект Montenegro Guidebook

Контентный сайт-справочник по Черногории на двух языках (**en — ведущий язык по умолчанию, на корне `/`**; ru → `/ru/`; uk не используется, удалён 2026-06-22): достопримечательности, города, еда (директория проверенных мест), маршруты, транспорт, аренда авто, страховки, развлечения, релокация/номады, новости. Места отбираются по высокому рейтингу (>4★ из нескольких источников) и сверяются с опытными путешественниками, гидами и местными жителями. Диверсифицированная партнёрская монетизация (экскурсии, жильё, аренда, страховки, eSIM). **Приоритет — EN/Tier-1** (западные туристы и номады), CIS (ru) — крепкий вторичный рынок (решения владельца — `docs/STRATEGY.md`). Единственный источник правды по продукту, нише и приоритетам — **SPEC.md**. Этот файл — правила работы внутри репозитория. При конфликте приоритет: SPEC.md → CLAUDE.md → STRATEGY.md → всё остальное.

> Форк рабочего движка **Georgia Guidebook** (страна-агностичная система переиспользована целиком). Стратегия и рынок Черногории — `docs/STRATEGY.md`, рыночная фактура — `docs/MONTENEGRO-ANALYSIS-2026-06-20.md`. Грузинские шаблоны-образцы — `docs/_georgia-reference/`.

## Стек и принципы
- **Astro 6 + Content Collections**, полная статика (SSG). Никаких SSR, БД, CMS.
- **Tailwind CSS 4**. Дизайн-токены — только из `src/styles/tokens.css` (SPEC §10). Хардкод hex-цветов в компонентах запрещён.
- **Без React.** Интерактив — нативный JS внутри Astro-компонентов, и только там, где без него нельзя: карта Leaflet, мобильное меню, переключатель языка.
- Хостинг: **Cloudflare Workers (Static Assets)**. Деплой = push в `main` (Workers Build: `npm run build` → `wrangler deploy`). Превью-деплои на ветках.
- Изображения: webp/avif через `astro:assets`, lazy-load, каждая ≤ 200 КБ.

## Команды
| Команда | Что делает |
|---|---|
| `npm run dev` | dev-сервер |
| `npm run build` | прод-сборка; обязана проходить без ошибок и предупреждений |
| `npm run check` | `astro check`: типы и схемы контента |
| `npm run lint` / `npm run format` | ESLint / Prettier |
| `npm run test:links` | проверка внутренних ссылок и /go/-редиректов |
| `npm run qa` | финальный GO/NO-GO: чистая сборка + гейты + статический аудит `dist/` |

Перед каждым коммитом: `npm run build && npm run check` — оба зелёные (для контентных правок — `npm run qa` = GO).

## Железные правила
1. **Языки (EN-first).** Сайт двуязычный: **en + ru** (uk не используется, удалён 2026-06-22). Материал существует на **en** (ведущий язык, пишется первым и задаёт приоритет/ключи) + перевод **ru** с одинаковым slug. Половину пары en/ru не публиковать (взаимные hreflang, см. `check-parity`). Обновление фактов = правка обеих версий в одном коммите. ru — крепкий вторичный рынок, но приоритет и подбор тем — от EN-интента (`docs/STRATEGY.md`).
2. **Партнёрские ссылки** — только через `/go/{partner}?c={slug}` с `rel="sponsored nofollow noopener"`. Прямые URL партнёрок в контенте запрещены. На статью 1–3 AffiliateBox, не больше. Сегментация по языку (SPEC §16): EN → SafetyWing / DiscoverCars / GetYourGuide / Booking; CIS (ru/uk) → EKTA / Localrent / Aviasales.
3. **URL не меняются** после индексации. Ошибся в slug — 301 в `public/_redirects`, а не переименование.
4. **Факты не выдумывать.** Цены, адреса, телефоны, условия — только из брифа `briefs/{slug}.md` или надёжного источника. Нет данных → оставить `<!-- TODO: факт от владельца -->` и сказать об этом, а не сочинять. Рейтинги/отзывы из внешних источников (Google, TripAdvisor) дословно не перепечатывать — это нарушает их ToS; использовать как критерий отбора и ссылку, добавляя собственное описание (SPEC §15). **YMYL (визы/налоги/ВНЖ/страховка):** только официальные источники (gov.me / MUP / PwC) + видимая дата проверки + дисклеймер; не печатать меняющиеся правила как факт (напр. порог дохода номад-визы — формулой, не фикс-числом).
5. **Перф-бюджет (мобайл):** LCP < 2.0 с, CLS < 0.1, JS на странице < 50 КБ. Любая новая зависимость тяжелее 10 КБ — спросить владельца.
6. **Доступность:** контраст ≥ 4.5:1, видимый focus, alt у каждого изображения, рабочая навигация с клавиатуры.
7. **SEO-обвязка каждой страницы:** title ≤ 60 зн., description ≤ 155, canonical, взаимные hreflang en↔ru (x-default → en), OG-картинка, schema.org по типу шаблона (SPEC §14).
8. **Сторонние скрипты:** только одна аналитика (SPEC §17). Никаких виджетов соцсетей, онлайн-чатов, тег-менеджеров. Дисплейная реклама на старте НЕ подключается (конфликт CSP/перф); пересмотр при доле Tier-1 ≥40% (`docs/STRATEGY.md`).
9. **Доки не отстают от кода (правило владельца — обязательно).** После КАЖДОЙ доработки и ДО коммита синхронизировать документацию: при изменении продукта/правил — `SPEC.md` / этот `CLAUDE.md` / `docs/STRATEGY.md`; статус/прогресс — `PROGRESS.md`/`HANDOFF.md` (когда заведены); авто-память + её зеркало `docs/memory/`. **Доработка без актуализации доков и без коммита+пуша в `main` НЕ считается завершённой** (память `always-commit-push-and-update-docs`).

## Контент-конвейер (SPEC §12)
`briefs/{slug}.md` (ключевой запрос — **EN-first**, вопросы читателя, ФАКТЫ владельца, цены, фото) → **скелет:** `npm run new -- <тип> <slug> --title "…"` (генерит en/ru + папку фото + DoD-чеклист в комментарии) → черновик en по шаблону → фото (CC/свои) → перевод ru → **fact-check → content-editor** → пара коммитится вместе, draft снимается. Порядок именно такой: сначала факты сверяются (`fact-checker`), потом перевод, потом финальный приёмщик (`content-editor`).

## Definition of Done страницы
- `build` и `check` зелёные; Lighthouse mobile: Performance / SEO / Accessibility ≥ 90
- en на месте + перевод ru, hreflang взаимные, schema.org проходит Rich Results Test
- дата обновления видима; AffiliateBox ведёт на рабочий /go/-редирект
- внутренние ссылки: минимум 2 на связанные статьи; хаб раздела ссылается на статью

### DoD по типам контента (gate `npm test` → check-photos + check-interlinks ≥2; полный список — в скелете `new-content.mjs`)
- **Статья/Новость:** cover (CC c `coverCredit`/своё) + ≥3 инлайн-figure ИЛИ ≥6 gallery — **фото обязательны**; факты только из брифа/источника (`sources`), иначе TODO; `visit` только с источником+`checkedAt`; ≥2 внутр. ссылки; title ≤60, desc ≤155; hreflang en↔ru; 1–3 AffiliateBox через `/go/`.
- **Маршрут:** cover обязателен; `days/distanceKm/budgetFrom/stops` заполнены; после `stops` — `node scripts/build-route-geometry.mjs`; далее как статья.
- **Место (ресторан):** отбор по консенсусу >4★ из нескольких источников + наш `ratingNote/review` своими словами (чужие отзывы дословно нельзя, §15); cover обязателен; `geo.coord`+`mapUrl`; `hours/dishes` только из источника; `verifiedAt`=дата.
- **Услуга:** `name/summary/rubric/city`; cover опц. (директория — текст-карточка); `url/contact` реальные (без `example.com`); `sponsored:true` только платное; `demo:true` только для честного примера (бейдж «Пример», noindex).
- **Город:** `name/region/coord/summary`; cover обязателен; связать с путеводителем (`article: reference`) при наличии.

## Обновления формата (согласовано с владельцем)
- **FAQ убран со всего сайта.** Нет FAQBlock и FAQPage-schema (пункт DoD про FAQPage больше не применять). Не возвращать без явного решения владельца; новую инфо вплетать в тело статьи.
- **Фото — инлайн в тексте + мини-галерея внизу.** Ключевые фото = raw `<figure class="figure">` в .md с подписью «что на фото» + CC-кредит (width/height → 0 CLS, lazy); остальные — во frontmatter `gallery`. Каждое фото один раз. Цель: ключевые направления (Котор/Будва) ~20, города ~12–15 «по качеству».
- **Цена и часы — поле `visit`** (`price`/`hours`/`note`/`checkedAt`) → VisitInfo. Валюта Черногории — **евро (€)**. Цены/часы не выдумывать (правило 4): только надёжный источник + дата проверки + «уточняйте»; нет данных → не указывать.
- **Порядок в хабах — `featuredOrder`** (меньше = выше). Флаг Черногории в хедере — ассет `/images/me-flag.svg`.
- **Переключатель языка убран из строки шапки** (решение владельца 2026-06-21) — теперь внутри выпадающего меню (кнопка ☰) внизу, с подписью-меткой; в подвале (Footer) дублируется как запасной путь. В основной строке шапки только бренд + разделы + кнопка-меню.
- **Favicon/иконки = флаг Черногории** (2026-06-21, решение владельца). Был грузинский флаг в `public/favicon.svg` (хвост движка) → заменён на квадратный флаг Черногории (красное поле, золотая рамка + двуглавый орёл, как `me-flag.svg` в шапке). Перегенерены и растровые: `public/icons/{favicon-96,apple-touch-icon-180,icon-192,icon-512,icon-512-maskable}.png` (maskable — full-bleed, рамка убрана). В `<head>` добавлен PNG-фолбэк `favicon-96.png` (Google-выдача/старые браузеры). Источник генерации — `scripts/_make-icons.mjs` (gitignored). Google покажет иконку после переобхода живого домена.
- **Аналитика GA4 подключена** (2026-06-21, `G-MMFK991W8V`) — единственная аналитика (правило 8). Внешний self-скрипт `public/js/analytics.js` (script-src 'self', §18), грузит gtag с googletagmanager.com; шлёт данные только с прод-домена (guard по hostname). CSP (`public/_headers`) расширен доменами google-analytics.com/googletagmanager.com. Measurement ID — публичный (виден в исходнике), не секрет.
- **Языки (EN-first):** сайт 2-язычный — **en (ведущий, корень), ru (`/ru/`)**. uk удалён 2026-06-22 (старые /uk/-URL → 301 на корень, `public/_redirects`). en пишется первым и задаёт приоритет; ru — перевод (крепкий вторичный рынок). Прочие языки не делаем. Статус/архитектура — `docs/STRATEGY.md`.
- **Гео (РЕШЕНО):** регионы — 3 макрорегиона (`coastal`/`central`/`northern`); базовые города — Podgorica, Budva, Kotor, Tivat, Herceg Novi, Bar, Ulcinj, Cetinje, Nikšić, Žabljak, Kolašin; точки «как доехать» (accessFrom) — Podgorica/Tivat/Budva; города директории еды — Budva/Kotor/Podgorica. Менять — только синхронно в `content.config.ts` + `i18n/types.ts` + словарях (gate `check-enums`).
- **Лучшие фото + развёрнутое описание (правило владельца).** При добавлении ЛЮБОЙ статьи/раздела — только самые качественные фото (CC/свои, отбор глазами; слабые/мусорные кадры не публиковать) и развёрнутое описание (объём и структура: инлайн-фигуры с подписями + мини-галерея, выверенные факты с источником, «как доехать», visit). Не тонкие «заглушки на отвяжись».
- **Синхронизация доков после каждой доработки (правило владельца).** Завершив доработку — обновить продуктовые доки (`SPEC.md`/`CLAUDE.md`/`docs/STRATEGY.md`) при изменении правил; `PROGRESS.md`/`HANDOFF.md` (когда заведены) — снимок прогресса; авто-память + её зеркало `docs/memory/`. Доки не должны отставать от кода. Финал доработки — коммит + пуш в `main`.

## СТАНДАРТЫ (объём, фото, тон)
Единые нормы публикации. При появлении хаб-файла `docs/STANDARDS.md` — он первичен; формулировки держать едиными, без расхождений.
- **Объём:** статья — **1200–2000 слов** (первый абзац — прямой ответ за 2–3 предложения; далее H2/H3 под вопросы читателя; без «воды», без FAQ-блока). Новость — **1500–2000 знаков** (~250–350 слов, 3–5 разделов, **≥4 внутренних ссылки**).
- **Фото (минимум):** статья — **≥5** (cover + ≥4 инлайн); новость — **≥2** (cover + ≥1); маршрут — cover + фото **каждой остановки**; город — **≥10**; столица — **≥15**. Каждое фото уникально на материал; формат **webp ≤200 КБ**; источник — своё или **CC/PD (Wikimedia/Openverse/Unsplash) с атрибуцией** (автор + лицензия в `coverCredit`/`figcaption`). _Действующий гейт `check-photos` пока требует cover + ≥3 инлайн (или ≥6 gallery) — qa зелёный; ужесточение до ≥5 и добор существующих статей (сейчас cover + 3) — в `ROADMAP-FIX.md`._
- **Город = статья `category=cities`** (английский слаг; коллекция `cities` НЕ используется — пуста во всех форках). _(В движке-доноре звалось `goroda`; у нас англо-слаги.)_
- **Тон:** живой, человечный, как у носителя языка; НЕ ИИ-генерёнка; факты — **только из источника/брифа**; даты/время/числа — **только из источника текущего года** (для ежегодных событий прошлогодние даты не копировать).
- **Партнёрки — только через `/go/{partner}`** с `rel="sponsored nofollow noopener"` (правило 2); прямые партнёрские URL в контенте запрещены.

## Чего НЕ делать
- Не добавлять React, CMS, базы данных и платные сервисы без явного решения владельца.
- Не клонировать инструменты (my_agents, ui-ux-pro-max) внутрь этого репозитория — они подключаются как плагины Claude Code, см. SETUP-GUIDE.md.
- Не генерировать массово страницы без уникальной фактуры: «SEO-генерёнка» = риск бана и прямое нарушение SPEC §15.
- Не трогать логику `/go/` без обновления таблицы партнёрок в SPEC §16.
- Не выдумывать факты по Черногории (особенно YMYL — визы/налоги/страховка); не противопоставлять «дешевле Албании» (паритет — `docs/STRATEGY.md`).

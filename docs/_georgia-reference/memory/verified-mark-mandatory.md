**Правило владельца (2026-06-18):** все заведения (рестораны `/eda/`) и услуги (`/relokatsiya/uslugi/`), которые МЫ вносим, ОБЯЗАНЫ нести видимую метку **«Проверено»** (галочка ✓).

**Why:** ценность директории = «проверено нами» (обещание «проверенных мест» + сигнал доверия/E-E-A-T); без метки заведение неотличимо от непроверенного.

**How to apply:**
- Единый знак = существующий компонент **`VerifiedBadge`** (вино-галочка ✓ + «Проверено · {дата}», дата из `verifiedAt`; опц. расшифровка «опытными путешественниками, гидами и местными»). **НЕ изобретать новых галочек** — один знак доверия на весь сайт (консистентность UI).
- **Честность (правило 4):** `verifiedAt` ставить ТОЛЬКО при реальной сверке (данные сверены с источниками/на месте). «Проверено» на непроверенном/вымышленном — запрещено.
- **Рестораны** (`/eda/`): `verifiedAt` обязателен схемой, но в карточке — **компактный бейдж «Проверено» БЕЗ даты** (решение владельца 2026-06-18: даты не показываем НИКОГДА; `verifiedAt` лишь хранится). **Услуги** (`/uslugi/`): ✅ РЕАЛИЗОВАНО 2026-06-18 — добавлено поле `verifiedAt` в схему `services` + рендер **компактного** `VerifiedBadge` (проп `compact`) в ServicesDirectory; ставить только реальным (демо — без). `VerifiedBadge` получил `compact`-вид (✓ + «Проверено», без даты) + i18n `verified.short`.
- **Витрина** (заведения на главной): компактная метка ✓ на карточке (когда заведения туда попадут).
- Заведения добавляю я — процедура в `CONTENT_GUIDE.md` §12–13.

**Дизайн/UI-UX (анализ 2026-06-18):** для плотных карточек (директория услуг, витрина) — КОМПАКТНЫЙ вариант VerifiedBadge (✓ + «Проверено», без даты/расшифровки); полный (с датой + расшифровкой) — на странице места/в шапке статьи. Цвет — брендовая **вино-галочка** (консистентно с сайтом), не зелёная/синяя. Размещение — у названия заведения. Один знак на карточку, не дублировать со спонсор-рамкой/бейджами. См.

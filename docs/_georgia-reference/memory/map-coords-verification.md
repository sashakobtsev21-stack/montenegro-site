**Как работает карта в статье:** Leaflet-карта рендерится, когда в frontmatter есть блок `geo:` с `coord: [lat, lng]` (+ опц. `address`); `map-init.js` делает `setView(coord, 16)` для одиночной точки. **Чтобы УБРАТЬ карту — удалить весь блок `geo`** (3 строки geo/coord/address). `sed -i '/^geo:$/,+2d' файл` — но правит во всех 3 языках (ru/uk/en).

**Решение владельца (2026-06-19): карту убрали** из `kazino-batumi`, `kazino-tbilisi`, `rynki-tbilisi`, `plyazhi-batumi` — там одна точка не имеет смысла (казино разнесены по городу, пляжи по побережью, рынки в 2 местах). `teatro-batumi` — НЕ трогать (карта оставлена по указанию владельца).

**🔴 ГЛАВНЫЙ УРОК — координаты сверять ВЕБОМ, не памятью:** при аудите карт я по памяти пометил 4 точки как «подозрительные» (Греми/Лагодехи/Кацхи/Гори) — после веб-проверки ВСЕ ЧЕТЫРЕ оказались ВЕРНЫ (совпали с Wikipedia до знаков); мои «исправления по памяти» сломали бы их. Память LLM на координаты ненадёжна. Проверять каждую точку через `WebSearch "<landmark> coordinates latitude longitude"` против Wikipedia (поле coordinates) / OSM. Грабля геокодеров: latitude.to/geodatos иногда дают неверную точку (напр. Уплисцихе, Вардзия) — доверять Wikipedia DMS.

**Итог аудита 48 статей с картами (2026-06-19):** датасет очень аккуратный (координаты явно сорсились с Wikipedia, масса точных совпадений). Реально неверными были только **3**: `kutaisi` 42.25/42.7 → 42.2679/42.6946 (~2 км южнее центра), `cherepashje-ozero-tbilisi` 41.7092/44.7589 → 41.7003/44.7542 (Черепашье озеро, ~1.1 км), `fabrika-tbilisi` 41.7089/44.7969 → 41.7097/44.8025 (~0.5 км). Остальные верны. Коммиты: убраны карты + фиксы — `6b7b914`.

**Процесс проверки:** `_scratch/news/coords.mjs` парсит frontmatter всех `src/content/articles/ru/*.md` → таблица slug/coord/title (флаг «вне границ Грузии» lat 41–43.6 / lng 40–46.8).

Создай новую статью для сайта на ДВУХ языках: EN (ведущий) + RU.
Тема и вводные от меня: $ARGUMENTS

Строго следуй CONTENT_GUIDE.md (Content Collections, frontmatter, фото,
ссылки, hreflang, гейты). Действуй так:

1. Если в моих вводных ($ARGUMENTS) не хватает чего-то из списка ниже —
   СНАЧАЛА задай мне уточняющие вопросы одним сообщением, не начинай писать:
   - целевой поисковый запрос на EN (как ищут в Google) — EN ведущий, задаёт приоритет
   - кластер и тип (TRAFFIC трафик / MONEY деньги)
   - есть ли у меня личные факты (цены, конторы, тайминги) или собирать из интернета
   Запрос на RU подбери сам через веб-поиск (как реально ищут носители,
   НЕ калька с английского), slug сгенерируй сам латиницей по теме.

2. Когда данных достаточно — напиши две версии (en — первой, ведущей):
   - Факты (цены, цифры, расстояния, названия) ИДЕНТИЧНЫ в обеих версиях.
   - НЕ дословный перевод. EN — как пишет носитель (ведущая версия). RU —
     естественной речью, не калька. У каждой версии свой H1 с ядром своего запроса.
   - H1 (title) ≤ 60 символов (гейт). Длинный хвост запроса — в подзаголовки.
   - Оба файла с ОДИНАКОВЫМ slug (для парности и hreflang).
   - Используй веб-поиск для актуальных данных 2026. Факты не выдумывай.

3. Фото по правилам CONTENT_GUIDE.md:
   - сначала переиспользуй существующие в public/images/;
   - своих нет — оставь <!-- ФОТО: что нужно — вставлю сам -->;
   - сток только Unsplash/Pexels конкретными ссылками по теме;
   - webp ≤200КБ, alt на языке версии.

4. Внутренние ссылки (мин. 2 — гейт): на статьи того же языка, спица↔опорная,
   TRAFFIC→MONEY. Партнёрские — только через /go/.

5. В конце прогони `npm run qa` и добейся GO (почини гейты: парность, H1≤60,
   мин. 2 ссылки, лимиты title/desc). Покажи мне обе версии и результат qa.
   Коммить и пушь ТОЛЬКО после моего «ок».

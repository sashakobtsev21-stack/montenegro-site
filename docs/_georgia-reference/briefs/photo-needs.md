# Бриф: фото и цены блюд для ресторанов (что нужно и как прислать)

> Рабочий список для сбора фото И цен на популярные блюда. На сайт сам файл не
> попадает (briefs/). Пришлёшь по схеме → я конвертирую фото в webp ≤200 КБ,
> разложу по карточкам (обложка + галерея до 5), добавлю цены блюд, пропишу
> подписи на ru+uk, соберу и задеплою.

**Два вида данных по каждому ресторану:** (1) фото (раздел ниже), (2) цены на
5–10 популярных блюд (раздел «Цены на блюда»).

## Как это работает
1. Снимаешь/получаешь фото (см. правило о правах ниже).
2. Называешь файлы по схеме `{slug}-N` (slug — в таблицах ниже).
3. Присылаешь — я всё оптимизирую, переименую под сайт и вставлю. Тебе НЕ нужно
   жать или конвертировать самому.

## Правила именования
- **`{slug}-1`** — главное фото (обложка карточки). Лучше **горизонтальное**.
- **`{slug}-2` … `{slug}-5`** — доп. фото для галереи-переключалки (по желанию, до 4 шт.).
- Пример для Barbarestan (slug `barbarestan`): `barbarestan-1.jpg`, `barbarestan-2.jpg`, …
- Формат **любой**: jpg / png / heic (с айфона — ок). Размер — **чем больше, тем лучше**
  (минимум ~1280 px по широкой стороне). Я сам сожму в webp ≤200 КБ.
- Если по месту несколько фото — можно прислать папкой с именем `{slug}`.

## ⚠️ Только фото, которые можно публиковать
- **Твои собственные** снимки, ИЛИ **фото от самого заведения с разрешением** опубликовать.
- **НЕЛЬЗЯ** брать из Google / TripAdvisor / соцсетей / чужих сайтов — это чужой копирайт
  (риск жалоб и бана сайта).
- По каждому фото скажи **источник** (твоё / прислал ресторан) — для корректной подписи.
  Своё фото места также даёт право на бейдж «Проверено».

## 1b. Цены на 5–10 популярных блюд (по каждому ресторану)
Для **каждого** заведения из таблиц выше нужны цены на 5–10 ходовых позиций.
Цены показываются в карточке как **ориентир** («ориентир, уточняйте на месте») —
точная сумма не обязательна, но цифра должна быть реальной, не выдуманной.

**Откуда брать (источник = правда):**
- меню самого заведения (бумажное/на их сайте/в их соцсетях) — лучший источник;
- спросить у заведения (в том же письме, где просим фото);
- агрегаторы (меню на их страницах) — только если совпадает с актуальным меню.

**Как прислать** — простым списком на каждое место (slug + позиции):
```
barbarestan:
  Хинкали — 2.5 ₾ (за шт.)
  Хачапури аджарули — 18 ₾
  Чахохбили — 24 ₾
  Лобио — 12 ₾
  Бокал вина — 9 ₾
```
- Валюта — лари (₾). Можно «за шт.» / «за порцию» в скобках, если важно.
- 5–10 позиций: лучше «визитные» блюда места + базовые (хинкали, хачапури, вино).
- Дата, на которую цены актуальны (для пометки «уточняйте» и ревизии).

Я внесу их в поле `dishes` карточки (название + цена), покажутся компактным
блоком «Цены на основные блюда». Цены/блюда **не выдумываю** (правило 4): нет
данных по месту — поле просто не показывается, пока не пришлёшь.

## 3. (Опционально) свои фото мест → бейдж «Проверено»
Достопримечательности и города уже с CC-фото, но если пришлёшь **свои** снимки —
поставим бейдж «Проверено · дата» (сильный сигнал доверия). Имя файлов = `{slug}` места
(slug виден в URL статьи, напр. `gergeti`, `batumi`, `mtskheta`). Приоритет —
флагманы: Тбилиси, Батуми, Кутаиси, Гергети/Казбеги.

## Куда лягут файлы (для разработчика — справочно)
- Рестораны: обложка → `public/images/restaurants/{slug}.webp`; галерея →
  `public/images/restaurants/{slug}/g1.webp … g4.webp`.
- Статьи/места: обложка → `public/images/{slug}.webp`; галерея/инлайн →
  `public/images/{slug}/gN.webp`.
- Каждое фото — webp ≤200 КБ, lazy, фикс. аспект (0 CLS); подпись/кредит — в `coverCredit`
  / `gallery[].credit` и alt на ru+uk. Обновлять обе языковые версии вместе.

Директория `/eda/` (`EdaDirectory.astro`) переработана 2026-06-16 по ресёрчу конкурентных гид-директорий еды (Infatuation, Wander-Lush, Eater, TripAdvisor) в рамках статики / CSP `script-src 'self'` / перф-бюджета.

**Что сделано (не откатывать без причины):**
- **Фильтры — чипы, а не выпадающие списки.** Фасеты: Город · Кухня · Ценник (рендерятся при ≥2 значениях). Чипы = `button[aria-pressed]`, множественный выбор внутри фасета (ИЛИ), И между фасетами. Логика — `public/js/eda-filter.js` (матч по `data-{facet}` на обёртке карточки). «Сброс» (`data-eda-clear`) появляется при активных чипах; живой счётчик + пустое состояние. Без JS все карточки видимы.
- **Блок «Как мы отбираем места»** (`.selection`, i18n `eda.selectionHeading`/`selectionNote`) вместо старой штриховой «образец»-плашки, выглядевшей недоделанной. Сигнал доверия (§15): отбор 4+★, своя редактура, спонсор помечен «Партнёр». Ключи `sampleNote*` ПЕРЕИМЕНОВАНЫ — не возвращать.
- **Карточка (`RestaurantCard`):** мета-строка район·кухня·ценник → бейджи-пилюли (`.rcard__badge`).
- Группировка по городам сохранена; city-чип быстро прячет неподходящие группы.

НЕ реализовано пока (осознанно): «Open now» (нужны структурированные часы — сейчас `hours` свободная строка, нельзя выдумывать §4) и фасет «тип заведения» (`placeType` без i18n-лейблов). Коммит `3c76e81`. См. [[inbound-business-listings-ia]], [[live-site-no-dev-placeholders]].

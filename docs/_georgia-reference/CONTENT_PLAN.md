## 3. Правила ведения

1. **Приоритет кластеров:** Планирование поездки (верх воронки) → Аренда авто (MONEY) → Маршруты → Достопримечательности + Города → Еда → Развлечения → Релокация. Сначала верх воронки + MONEY и связующие, релокацию — в конце.
2. **Темп:** 3–5 статей в неделю (пара ru+uk считается за одну тему).
3. **Связка трафик → деньги:** каждая TRAFFIC-статья (достопримечательность/город/маршрут) ссылается на MONEY-кластеры (аренда авто, страховка), и наоборот. Партнёрские ссылки — только через `/go/` (CLAUDE правило 2), 1–3 AffiliateBox на статью.
4. **Взаимные ссылки после новой статьи.** Добавил статью в кластер — сразу проставь встречные ссылки из опорной и соседей на неё (как сделали для спицы «аэропорт Кутаиси»). Кластер должен быть связан в обе стороны.
5. **Каждая тема — пара ru + uk** (+ en по возможности), одинаковый slug; пишем строго по `CONTENT_GUIDE.md`. Перед публикацией — `npm run qa` = **GO**, коммит пары одним коммитом.
6. **Без дублей/каннибализации.** Один запрос-ядро = одна статья. Прежде чем заводить тему, проверь, не занят ли запрос (см. как проверяли кластер аренды). Если занят — расширяй существующую или бери свободный подзапрос.
7. **Факты не выдумывать** (CLAUDE правило 4): цены/часы/адреса — из источника + «уточняйте» + дата; нет данных → TODO.

## 4. Как взять тему из Wordstat (кратко)

[Яндекс.Wordstat](https://wordstat.yandex.ru) — оценка частотности русских поисковых запросов. Как пользоваться для плана:

1. **Введи «семя»** темы — например `аренда авто грузия` или `что посмотреть в грузии`.
2. **Левая колонка** — что включает запрос (фразы с этим словом + частотность за месяц); **правая** — похожие запросы (соседние идеи для новых тем).
3. **Бери длинный хвост:** конкретные частотные фразы (`аренда авто аэропорт кутаиси`, `аренда внедорожника грузия`) — у них ниже частота, но выше интент и проще выйти в топ, чем по общему `аренда авто`.
4. **Определи интент:** транзакционный (аренда, страховка, бронь) → **MONEY**; справочный (что посмотреть, как добраться) → **TRAFFIC**.
5. **Кластеризуй:** группа близких запросов = одна статья (ядро в H1, хвост — в H2/H3). Синонимы НЕ разносим по разным URL (это каннибализация, правило 6).
6. **UK/EN Wordstat не покрывает** — для них смотри подсказки Google и «People also ask» (как делает `/new-article`): запрос носителя, а не калька с русского.
7. **Записывай выбранную тему сюда** в раздел 1/2 со статусом ⬜ план → 🟡 в работе → ✅ готово.

> Wordstat показывает спрос, но не гарантирует уникальность фактуры. Тема идёт в работу, только если по ней есть, что написать своими словами и с проверяемыми фактами (иначе — «генерёнка», риск бана, SPEC §15).

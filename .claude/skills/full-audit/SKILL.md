# /full-audit — полный аудит сайта (мультиагентный)

Цель — дать максимально полную, проверенную картину состояния сайта по ВСЕМ аспектам и приоритизированный план. Источник правды по продукту — `SPEC.md`; правила — `CLAUDE.md`; статус — `PROGRESS/HANDOFF/ROADMAP`. Ничего не правим в коде в рамках аудита (только отчёты) — фиксы выполняются отдельно по итоговому плану.

## Фаза 0 — Подготовка (основной чат, ОДИН раз; не параллелить сборку)
1. `npm run qa > _scratch/audit/qa.txt` — чистая сборка + статический аудит dist (SEO/hreflang/schema/a11y/перф/smoke/CSP, GO/NO-GO).
2. `npm run qa:browser > _scratch/audit/qa-browser.txt` — mobile-Lighthouse (a11y/seo/bp/perf) + responsive (0 переполнений 320–1280).
3. Скриншоты ключевых шаблонов (мобайл+десктоп) → `_scratch/audit/shots/` (Playwright/системный Chrome, как `qa-responsive.mjs`).
   Все артефакты — в `_scratch/` (gitignored, вне `public/` → не попадают в dist/деплой).

## Фаза 1 — Аудит (параллельно, спец-агенты, read-only; НЕ пересобирать)
Каждый возвращает структурированные находки (severity/where/evidence/fix) + score измерения:
- `seo-engineer` — тех-SEO (title≤60/desc≤155/canonical/взаимные hreflang+x-default/schema.org/sitemap/robots/OG).
- `seo-strategist` — покрытие ниши, перелинковка, воронка справочная→деньги (§14), тактика против AI Overviews.
- `ui-ux-auditor` — иерархия/консистентность/навигация/мобайл/CTA/читабельность/состояния/доверие (+ скриншоты Read'ом).
- `accessibility-auditor` — WCAG: контраст vs токены §10, focus-visible, ARIA, reduced-motion, клавиатура.
- `perf-diagnostician` — бюджет §15: LCP/CLS/INP, JS≤50КБ/стр, img≤200КБ, вес зависимостей.
- `content-editor` — DoD, анти-бан §15 (уник. факты, без генерёнки), тон, дубли методологии.
- `reviewer` — корректность/типы/паттерны/запреты CLAUDE.
- `architect` — IA/URL (§7), границы модулей, контент-модель (§11).
- `security-auditor` — CSP/_headers, open-redirect /go/, секреты, CVE (§18).
- `tester` — ссылки/schema/паритет ru-uk-en/гейты (§21).
- `monetization-strategist` — рекламный инвентарь, /go/+rel, ненавязчивость (§16).

## Фаза 2 — Обсуждение (агенты спорят)
Панель кросс-ревью читает ВСЕ отчёты фазы 1 → дедуп пересечений, разрешение конфликтов между измерениями (например SEO «больше контента» ↔ perf «меньше веса», UX ↔ монетизация), адверсариальная проверка критичных находок (skeptic пытается опровергнуть — слабые отсеиваются), переранжирование по severity×impact×усилие.

## Фаза 3 — Синтез (агенты пишут итоги)
Сводный отчёт: executive summary + статус по измерениям (score) + подтверждённые находки по приоритету + разрешённые конфликты + план действий (что чинить в каком порядке, усилие) → `AUDIT-FULL-<дата>.md`; обновить `AUDIT.md` (закрытое/новое). Решения, требующие владельца, вынести отдельным блоком.

## После
Отчитаться владельцу по-русски (память communicate-in-russian), без правок кода. Фиксы — отдельной задачей по плану. Синхронизировать доки/память (update-docs-after-each-change). Коммит артефактов аудита (отчёт-доки; `_scratch` не коммитим).

## Запреты
Не править код в рамках аудита. Не пересобирать dist в параллельных агентах (гонка) — сборка один раз в фазе 0. Не выдумывать (правило 4). Плагины не создавать (CLAUDE: внешние инструменты не клонируем в репо).
